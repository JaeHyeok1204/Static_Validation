Slide 06 — Data Model
핵심 데이터 모델
버전 관리 구조와 검증 데이터 타입
🗂️ 버전 관리 구조
versions: string[] // ["v1.0", "v1.1", ...]
versionedData: Record<number, VersionData>
  dashboardData: 전체 진척도, 일정 상태
  chartData: 날짜별 검출/분석 추이
  subsystemsList: 16개 서브시스템 (A~P)
  rulesList: MAB + MISRA 규칙별 위배수
  issuesList: 이슈 관리 목록
  risksList: AI 생성 리스크 목록
  timeEvaluationComponent: 소요시간
  reportsDraft: AI 보고서 초안
📏 정적 검증 규칙 체계
MAB 규칙: 115개 규칙 ID (MAB_AR_*, MAB_DB_*, MAB_JC_*, MAB_NA_* 등)

MISRA 규칙: 155개 규칙 ID (MISRA_AC_SLSF_*)

각 규칙은 16개 서브시스템별 위배 건수 매핑:
subsystemViolations: Record<string, number>
👤 사용자 데이터 모델
id: string // 사원번호
email: string // 이메일 (인증용)
password?: string // SHA-256 해시
name: string
birthDate: string // 생년월일
teamName: string
position: string // 직책
geminiApiKey?: string
